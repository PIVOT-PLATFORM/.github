# Support — PIVOT Platform

## Canaux de support

| Canal | Usage |
|-------|-------|
| [GitHub Discussions](https://github.com/PIVOT-PLATFORM/pivot-core/discussions) | Questions d'architecture, idées, discussion générale |
| [Issues pivot-core](https://github.com/PIVOT-PLATFORM/pivot-core/issues) | Bugs backend · Features backend |
| [Issues pivot-ui](https://github.com/PIVOT-PLATFORM/pivot-ui/issues) | Bugs frontend · Features UI |
| [Security Advisories](SECURITY.md) | Vulnérabilités uniquement — ne pas ouvrir d'issue publique |

## Avant d'ouvrir une issue

1. Vérifier que l'issue n'existe pas déjà (recherche dans les issues ouvertes et fermées)
2. Reproduire le bug sur la dernière version
3. Inclure les logs, étapes de reproduction, environnement

## Temps de réponse

Petite équipe — délais variables mais tout est lu. Pas de SLA garanti sur les contributions externes.
